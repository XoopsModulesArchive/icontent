Manuel d'utilsation - Administration des pages

 

Voici une copie d'écran typique de l'administration des pages :

 

I Premier tableau - "Dossiers choisis"

Le principe de ce tableau est rigoureusement le même que celui de l'administration des dossiers. Référez vous donc à la section "Administration des dossiers - iContent v4.0" pour plus de détails.

 

II Deuxième tableau - "Dossier courant"

Le titre de ce tableau indique dans quel dossier vous vous situez lors de votre navigation. En dessous se trouve le contenu du dossier courant avec, à droite de chaque élément, les informations le concernant ou, pour les pages, la liste des actions que vous pouvez effectuer sur la page en question. Pour les informations sur les dossiers reportez vous à la section "Administration des dossiers - iContent v4.0" pour plus de détails. Il est à noter que comme pour l'administration des dossiers, seul le contenu activé est affiché dans cette partie.

Voici la liste des commandes disponnibles pour chaque page :

  • Repère 1 : vous permet de modifier la page en ligne via un éditeur wysiwyg assez puissant à mon gout. Nous parlerons de cette partie dans la suite de ce manuel.
  • Repère 2 : vous permet de créer un raccourci vers la page si il n'est pas encore créé ou de modifier le raccourci vers cette page si il est déjà créé.
  • Repère 3 : vous permet de consulter la date de la dernière modification ou mise à jour de la page.

Note : vous rencontrerez une autre option dans cette partie lorsque vous ne vous trouverez pas à la racine du répertoire "icontent/inPages" là ou vous mettez vos documents. Cette option permet de faire de la page l'accueil de son dossier parent. En clair lorsque un visiteur cliquera sur le nom du dossier parent dans l'explorateur iContent il verra directement la page d'accueil s'afficher plutôt que le contenu du dossier proprement dit. De plus une autre option permettant d'updater manuellement la page est disponnible.

 

III Troisième tableau - "Actions"

Ce troisième tableau est relativement voir complètement similaire à celui de l'administration des dossiers. Il permet d'effectuer plusieurs actions sur une liste de pages.

  • Repère 4 : vous permet d'activer une liste de pages ou une seule page si vous le désirez.
  • Repère 5: vous permet de configurer une liste (ou une seule) de pages entre autres pour changer leur nom, leurs droits d'accès etc...
  • Repère 6 : vous permet de désactiver des pages déjà activées.

 

IV Activer une page

Pour activer une page vous devez au préalable activer tous ses dossiers parents. Reportez vous à la section "Administration des dossiers - iContent v4" pour savoir comment procéder. Ensuite une fois cette opération faite, rendez vous dans le dossier parent de la page et sélectionnez son nom dans le tableau "Actions" tout à sa gauche (Remarque : le nom de la page sera son nom physique sur le serveur). Cliquez ensuite sur "Activer les pages" pour activer la ou les pages séléctionnée(s).

 

V Configurer une page

Cette partie est identique ou presque à celle de l'administration des dossiers. Reportez vous donc à la section "Administration des dossiers - iContent v4" pour de plus amples informations.

Voici une copie d'écran du résultat de la configuration d'une page :

La configuration d'une (ou plusieures) page est similaire à celle d'un dossier. Il est à noter cependant que la partie "Page d'accueil" est en moins.

 

VI Désactiver une page

Rendez vous dans le dossier parent de la page que vous voulez désactiver puis dans le tableau "Actions" tout à sa droite. Sélectionnez la (ou les) page(s) que vous désirez désactiver (son nom sera celui que vous lui avez donné si vous l'avez modifié) puis cliquez sur "Désactiver les pages".

Note : une page ne peut, bien sûr, être désactivée seulement une fois activée au préalable. De plus désactiver une page ne supprimera pas le fichier que vous avez placé sur le serveur. Cependant lorsque vous désactivez une page, vous perdrez toutes les modifications que vous lui avez faite online soyez donc prudent.

 

VII Modifier une page online

Voici une copie d'écran typique de la modification d'une page :

Pour modifier une page allez dans son dossier parent puis dans la colonne à coté de son nom dans le tableau "Dossier courant" (image 1, repère 1) cliquez sur "Modifier nom de la page" vous verrez alors apparaitre l'interface de modification comme ci dessu. Une fois votre page modifiée cliquez sur "Modifier" tout en bas pour valider vos modifications.

Remarque sur l'insertion d'images : toutes les images que vous insererez grâce à cette interface (et non pas manuellement en tapant un <img src=".....">) se trouvent dans le répertoire "inPages/imagesLibrary". Vous deverez donc uploader dans ce dossier toutes les images que vous insérer online. Biensûr, il est vital pour le bon fonctionnement de iContent de ne pas supprimer le dossier "inPages/imagesLibrary".

 

VII Remarques utiles sur le traitement des pages par iContent v4

Depuis la v4 il n'est plus utile de supprimer les balise <head>.....</head>, <html>, </html> et enfin <body ....>. En effet iContent s'en charge pour vous. De plus par rapport à la v3 le fait que iContent ne modifie pas certaines urls à légèrement changé. Désormais toute url relative commencant par "inPages" ne sera pas corrigée et sera donc considérée, d'une certaine façon, comme une url absolue. "inPages" apparait désormais comme un mot réservé à iContent. Je vous conseille donc déviter de l'utiliser particulièrement pour des pages à la racine de "inPages".

Depuis la v4 les pages sont traitées puis compilées dans le dossier "compiled". Si il n'y a pas eu de modification ou de mise à jour de la page originale (qui ne sera donc pas modifiée vous l'aurez compris) le fichier compilé sera alors directement affiché sinon il sera automatiquement mis à jour. De cette manière iContent ne fais quasiment plus les modifications de la page en temps réel ce qui lui permet de supporter une charge beaucoup plus importante de visisteurs sans surcharger les ressources serveur.

Remarques : la prise en charge des liens et urls diverses est beaucoup mieux prise en charge depuis la v4 grâce à son tout nouveau compilateur cependant cependant les formulaires, contrôles activeX et les objets flash ne sont pas encore gérés ce qui ne devrait cependant ne pas trop tarder. Depuis cette version iContent est désormais dédié à la compilation des pages html donc seules ces pages peuvent être activées via l'admin.
